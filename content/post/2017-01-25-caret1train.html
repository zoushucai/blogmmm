1. 使用caret包进行自动参数调整

与其对模型的每个参数选择任意值，这样做不仅耗费时而且不科学，不如对搜索参数值的过程进行引导从而找出最优组合
caret包最核心的功能是提供了一个train()函数作为标准接口，为分类和回归任务训练233种(不断在增加)不同的机器学习模型
需要考虑的3个问题
+ 需要使用数据来训练那种机器学习模型？ + 那些模型的参数是可以调整的，他们调整的空间有多大？
+ 使用何种评价标准来评估模型从而找到最优的候选者？

可参考238个模型以及相关参数的详细信息以及每种模型的详细参数解释，可能需要翻墙。可解决以上前两个问题

列出一些常用的模型：

自动调整的目标是搜索候选模型的集合，由所有参数组合的矩阵或者网格的形式构成，caret默认对每个参数最多搜索3个可能值(由参数tuneLength控制)，假设一共有p个参数，这意味着\(3^p\)候选模型将会被测试,若某个参数所有的值都不超过3，则可能改变，例如：C5.0 模型，model和winnow参数值只能选两个(分别是tree与rules 和TRUE与FALSE)，那么网格大小不再是\(3*3*3=27\)，而是 \(2*2*3=12\)

上述第三个问题，选择一种方法从候选者中识别最好的模型，caret包支持大部分性能统计量，包括：准确度、Kappa值(用于分类器)、R方值、RMSE(用于数值模型) ,如果需要的话也可以使用代价敏感方面的度量模式,比如灵敏度、特异性、ROC曲线下面积(AUC)等等
在各种选项中，大部分默认值都是合理的，例如,在分类模型中使用自助法抽样的预测准确度来选择最优性能的优化器.

1.2 定制调参的过程

train 函数为模型训练函数，它会自动的使用最佳模型中的参数对所有数据建立模型（我们给定的参数中最好的参数）

train(x, y, method = "rf", preProcess = NULL, ...,
  weights = NULL, metric = ifelse(is.factor(y), "Accuracy", "RMSE"),
  maximize = ifelse(metric %in% c("RMSE", "logLoss", "MAE"), FALSE, TRUE),
  trControl = trainControl(), tuneGrid = NULL,
  tuneLength = ifelse(trControl$method == "none", 1, 3))
  1. method = “rf”，使用哪种模型
  2. preProcess 数据预处理过程（默认不处理）。 “BoxCox”, “YeoJohnson”, “expoTrans”, “center”, “scale”, “range”, “knnImpute”, “bagImpute”, “medianImpute”, “pca”, “ica” and “spatialSign”.
  3. metric = “Kappa” ,模型评估用到的统计量
  4. 概率值（RMSE）
  5. 回归： 卡方、RMSE
  6. 分类： Accuracy、Kappa
  7. trainControl(method="boot",selectionFunction = "best",.....) 指定一系列的配置选项，与train函数配合使用，这些选项考虑到诸如重抽样策略以及用于选择最佳模型的度量方法。
  8. method: 用来设置重抽样的方法 （见下表），每种方法都有各自的参数。
  9. selectionFunction : 设定一函数用来在各个候选者中选择最优的模型，共3个参数:
    • best (默认，简单的选择具有最好的某特定度量的候选者)
    • oneSE (选择最好性能标准差之内的最简单的候选者)
    • Tolerance (选择某个用户特定比例之内的最简单的候选者)
  10. 如文章开头所展示的那样，每个模型都有自己的参数，这是就需要设置train函数中的tuneGrid参数网络,网络的每一列表示模型中的一个参数，前面的用点. 作为前缀。数据框中的每一行代表一种特定参数值得组合。如果不想自己手工创建这样的数据框，也可以使用expand.grid() 函数来创建数据框
  • train中参数tuneGrid = NULL对应 调优参数网格expand.grid()
    • 可自己建立数据框，列名称应与拟合函数的参数相同
    • 可用expand.grid()函数建立，参数为模型参数前加点. 作为前缀，例如C5.0的参数有trials,引用expand.grid(.trials=c(1:3),.winnow=“FALSE”)
重抽样方法 方法名 额外的选项和默认值
保持抽样 LGOCV p = 0.75 (训练数据比例）
k 折交叉验证 C V number = 1 0 ( 折的数目）
重复 k 折交义验证 repeatedcv number = 10 (折的数口） repeats = 10(迭代的次数）
自助法抽样 boot number = 25 (重抽样迭代的次数）
0.632 自助法 boot632 number = 25 (重抽样迭代的次数）
留一交叉验证法 LOOCV 无

1.5、并行计算

有没有感觉每运行一个模型，感觉都很慢，那么train()函数其实可以采用并行的，不过我在并行的时候，时间只能减半，很奇怪（明明是4核）
因为在参数调优过程中的每一次建模之间是独立的。因此，理论上是可以并行的来做参数调优的。caret也提供了这一功能。只需要在train之前初始化并行环境即可。

采用上面的过程代码，不并行浪费的时间

rm(list=ls())
library(mlbench)
data(Sonar)#208个数据 一共61个变量,最后一个变量为目标变量class(因子变量：水平R、M)
##划分数据集和测试集
library(caret)
set.seed(998)
inTraining <- createDataPartition(Sonar$Class, p = .75, list = FALSE)
training <- Sonar[ inTraining,]
testing  <- Sonar[-inTraining,]

## 基本参数的设置----trainControl（）
fitControl <- trainControl(## 10次重复10交叉验证
                           method = "repeatedcv",
                           number = 10,
                           repeats = 10)
## 调优参数网格设置-----expand.grid()
## 默认为随机搜索 search = "random" 情况下 (tuneLength=3) 最多每个参数搜索3个值
gbmGrid <-  expand.grid(interaction.depth = c(1, 5, 9), 
                        n.trees = (1:30)*50, 
      #一共有90个参数设置，interaction.depth 3个 树的颗数n.trees = (1:30)*50 ，30种，一共90个参数
                        shrinkage = 0.1,
                        n.minobsinnode = 20)
set.seed(825)
unparallel_time=system.time({gbmFit2 <- train(Class ~ ., data = training, 
                 method = "gbm", 
                 trControl = fitControl, 
                 verbose = FALSE, 
                 ## Now specify the exact models 
                 ## to evaluate:
                 tuneGrid = gbmGrid)})

library(foreach)
